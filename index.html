Section 2
Why Anything, Why Now, Why Glean
Strategic rationale

Why Anything – An enterprise AI fabric for Flex

Falcon Flex is already a >$1.35B ARR engine growing >200% YoY, with the ambition to make it the standard licensing model while sustaining strong year‑over‑year growth and expanding operating margins. The current model—hiring more people and relying on heroic individual effort to manage increasing complexity—does not scale.

To avoid hitting an execution ceiling, CrowdStrike needs a true enterprise AI experience that:

  • Gives every employee a unified way to find, understand, and act on information.
  • Translates directly into measurable productivity gains and higher employee satisfaction.

Why Now – This planning cycle

Flex has clear product–market fit and is re‑accelerating net‑new ARR, but GTM and operations are still heavily dependent on:

  • Tribal knowledge scattered across tools and teams.
  • “Hero” efforts to build ad‑hoc internal tools and workflows.

As more Flex modules come online and CrowdStrike continues to scale globally, this model won’t sustain Flex growth ambitions. Now is the window to standardize what’s working and put a central AI layer in place before complexity outpaces current processes.

Why Glean – Strategic platform choice

Glean is an enterprise‑ready AI platform built for organizations like CrowdStrike:

  • Securely connects to 100+ core systems (Falcon Flex data, collaboration, CRM, ITSM, Dev tools, HR, etc.) while fully honoring existing security and permissions.
  • Provides a single, secure system of context for every employee: search, assistant, and agents in one place.
  • Lets CrowdStrike builders leverage advanced MCP and agent capabilities instead of reinventing foundational AI plumbing.

Based on the joint ROI modeling, this partnership is expected to unlock tens of millions of dollars in annual productivity savings that can be reinvested into higher‑value work, including Falcon Flex strategy and execution.

Recommended Plan – 2‑Year Glean Key Partnership
Primary option

3.1 Deployment & scope

  • Product: Glean EnterpriseFlex.
  • Hosting: Self‑hosted in CrowdStrike’s AWS environment.
  • LLM key model: Glean Key (Glean Universal Model Key).
  • Users: Contractual path to 12,000 licensed users aligned to Flex and adjacent functions.
  • Term: 2‑year initial term with built‑in price protection.
  • Support & services: Premium Support + Outcomes package focused on productivity, adoption, and ROI realization.

3.2 Commercial design principles

This option is intentionally structured to:

  • Respect current budget constraints in the first purchase year.
  • Phase spend over time while still delivering an end‑state 12k‑user deployment.
  • Concentrate Year‑1 spend where the pilot has already shown the highest value and adoption.
  • Lock in unit economics for the full 12,000‑user footprint and avoid re‑opening core pricing as Flex scales.

3.3 What CrowdStrike gets with Glean Key

Model performance & choice

  • Glean centrally manages a universal LLM key across Azure OpenAI, Vertex, Bedrock, and OpenAI.
  • Optimized for best price–performance and reliability.
  • New models and agentic features land here first.

Operational simplicity

  • Glean owns capacity planning, routing, failovers, and incident response for LLM usage.
  • Reduces operational load for infra and security teams.

Predictable economics

  • Seat‑based EnterpriseFlex licensing.
  • Pooled credit model for advanced agents and Deep Research.
  • Included credits sized to CrowdStrike’s adoption plan; overages only if usage materially exceeds expectations.

Section 4
Alternative Plan – Self Key (Reference Only)
Self‑managed LLM option

4.1 Self Key option overview

The Self Key construct keeps the same 12,000‑user end‑state but shifts runtime responsibilities and some economics to CrowdStrike.

  • Product: Glean EnterpriseFlex.
  • Hosting: Self‑hosted in CrowdStrike’s environment.
  • LLM key model: Self Key (CrowdStrike‑provided LLM key).
  • Users: Same 12,000‑user end‑state as Glean Key, with separate economics.

4.2 What CrowdStrike owns under Self Key

  • Procures, provisions, and manages LLM capacity across vendors and regions.
  • Owns routing, failovers, and capacity planning for LLM traffic.
  • Carries more operational load across infrastructure, security, and procurement.

4.3 Tradeoffs vs. Glean Key

Self Key – when it may be preferred
Reference construct
  • More direct control over underlying LLM vendor(s) and tenancy.
  • Discounted seat and credit economics to reflect infra costs borne by CrowdStrike.
  • Flexibility to shift models or vendors based on internal AI strategy.

Key considerations

  • Operational overhead increases (capacity, reliability, upgrades, troubleshooting).
  • Certain advanced features may lag or require additional alignment.
  • All LLM/model spend is funded and forecast by CrowdStrike, making true TCO more variable over time.

The proposal keeps Self Key as a viable alternative, but the primary recommendation and ROI modeling focus on Glean Key, where value realization is highest and complexity lowest.

Section 6
Implementation & Joint Operating Model
From pilot to fabric

6.1 Deployment approach

  • Self‑hosted + Glean Key in CrowdStrike’s AWS environment.
  • Joint architecture and security review leveraging work completed during the pilot.
  • Phased rollout to:
    • Falcon Flex core teams and supporting functions.
    • Broader enterprise cohorts as value and adoption are proven.

6.2 Outcomes program

The Outcomes package is explicitly tied to:

  • Realizing the search & assistant productivity assumptions in the ROI model.
  • Defining and tracking Flex‑specific KPIs (e.g., time‑to‑answer, time‑to‑first‑draft, number of Flex deals supported).
  • Enabling GTM agent use cases (Account 360, proposals/RFPs, competitive intel) once the foundation is stable.

Joint operating cadence

  • 1
    Executive alignment
    Confirm Falcon Flex objectives and the small set of metrics that serve as the joint success scorecard.
  • 2
    Technical deep‑dive
    Finalize self‑hosted architecture, security posture, and data‑source coverage using learnings from the pilot.
  • 3
    Wave‑based rollout
    Start with Flex‑critical teams and adjacent functions, then expand to the full 12,000‑user footprint as value is proven.
  • 4
    Value realization & expansion
    Track ROI realization vs. the model, then layer in additional use cases (agents, ticket deflection, GTM workflows) to compound value.

Under Self Key, CrowdStrike also funds all LLM/model usage directly. Actual all‑in TCO depends on model choices, agent usage, and vendor pricing over time, making Glean Key structurally more predictable.
