Section 2
Why Anything, Why Now, Why Glean
Strategic rationale

Why Anything – An enterprise AI fabric for Flex

Falcon Flex is already a >$1.35B ARR engine growing >200% YoY, with the ambition to make it the default licensing model while continuing to expand operating margins. The current model—hiring more people and relying on heroic individual effort to manage increasing complexity—does not scale.

To avoid hitting an execution ceiling, CrowdStrike needs to deliver a true enterprise AI experience that:

  • Gives every employee a unified way to find, understand, and act on information.
  • Translates directly into measurable productivity gains and higher employee satisfaction.

Why Now – This planning cycle

Flex has clear product–market fit and is re-accelerating net-new ARR, but GTM and operations are still heavily dependent on:

  • Tribal knowledge scattered across tools and teams.
  • “Hero” efforts to build ad-hoc internal tools and workflows.

As more Flex modules come online and CrowdStrike continues to scale globally, this model will not sustain Flex growth ambitions. Now is the window to:

  • Standardize and codify what’s working before complexity outpaces current processes.
  • Put a central AI layer in place that elevates every employee’s day-to-day work while Flex scales globally.

Why Glean – Strategic platform choice

Glean is an enterprise-ready AI platform designed for organizations like CrowdStrike:

  • Connects securely to 100+ core systems (Falcon Flex data, collaboration, CRM, ITSM, Dev tools, HR, etc.) while fully honoring existing security and permissions.
  • Provides a single, secure system of context for every employee: search, assistant, and agents in one place.
  • Enables CrowdStrike builders to leverage advanced MCP and agent capabilities instead of reinventing foundational AI plumbing.

Based on joint modeling in the ROI workbook, this partnership is expected to unlock tens of millions of dollars in annual productivity savings that can be reinvested into higher-value work, including Falcon Flex strategy and execution.

3.2 Commercial design principles

This option is intentionally structured to:

  • Respect current budget constraints in Year 1.
  • Phase spend over time while still delivering an end-state 12k-user deployment.
  • Concentrate Year 1 spend where the pilot has already shown the highest value and adoption.
  • Lock in unit economics for the full 12,000-user footprint and avoid re-opening core pricing as Flex scales.

3.3 What CrowdStrike gets with Glean Key

Model performance & choice

  • Glean centrally manages a universal LLM key across providers (Azure OpenAI, Vertex, Bedrock, OpenAI).

  • New models and advanced agent features land here first.

Operational simplicity

  • Glean owns capacity planning, routing, failovers, and incident response for LLM usage.
  • Reduced operational load for infra and security teams.

Predictable economics

  • Seat-based EnterpriseFlex licensing.
  • Pooled credit model for advanced agents and Deep Research.
  • Included credits sized to CrowdStrike’s adoption plan; overages only if usage materially exceeds expectations.

Section 6
Implementation & Joint Operating Model
From pilot to enterprise fabric

6.1 Deployment approach

  • Self-hosted + Glean Key in CrowdStrike’s AWS environment.
  • Joint architecture and security review, leveraging the work completed during the pilot.
  • Phased rollout to:
    • Falcon Flex core teams and supporting functions.
    • Broader enterprise cohorts as value and adoption are proven.

6.2 Outcomes program

The Outcomes package is explicitly tied to:

  • Realizing the search & assistant productivity assumptions in the ROI model.
  • Defining and tracking Flex-specific KPIs (e.g., time-to-answer, time-to-first-draft, number of Flex deals supported).
  • Enabling GTM agent use cases (Account 360, proposals/RFPs, competitive intel) once the foundation is stable.

Joint cadence & governance

  • 1
    Executive alignment
    Confirm Falcon Flex objectives and the small set of metrics that will serve as the joint success scorecard.
  • 2
    Technical deep-dive
    Finalize self-hosted architecture, security posture, and data-source coverage using learnings from the pilot.
  • 3
    Wave-based rollout
    Start with Flex-critical teams and adjacent functions, then expand to the full 12,000-user footprint as value is proven.
  • 4
    Value realization & expansion
    Track ROI realization vs. model, then layer in additional use cases (agents, ticket deflection, GTM workflows) to compound value.
